# Conditional SliceGAN — 설계 개요

이 문서는 조건부 SliceGAN을 **왜, 무엇으로, 어떻게** 만드는지 쉬운 말로 정리합니다.

---

## I. 왜 만드는가 — 문제 정의

### 1. 기존 SliceGAN의 한계
- 2D 단면 이미지 1장을 받으면, 그 이미지 느낌대로 3D를 만드는 생성기를 **처음부터 학습**.
- 다른 구조가 필요하면? → 모델을 **처음부터 다시 학습**.
- 즉 **구조 1개 = 모델 1개**. 재사용 불가.

### 2. 우리가 바라는 것
- 모델을 **딱 한 번만** 학습.
- 추론 시점에 "이렇게 생겨라"는 조건 — 이미지 **1장 이상** — 을 넣는다.
- 모델이 그 조건을 따르는 3D를 출력. 조건이 바뀌면 출력이 바뀜. **재학습 불필요.**

### 3. 현실의 데이터 제약
- 조건 이미지를 많이 넣을수록 원하는 구조에 가까운 결과.
- 하지만 모든 단면을 얻을 수 있으면 → 애초에 SliceGAN이 불필요.
- 현실적으론 FIB 같은 기법으로 **일부 단면만** 띄엄띄엄 얻을 수 있음.
- **목표**: 최소한의 단면만 주고 사이를 모델이 채우게 한다.

> **한 줄**: "구조마다 모델 따로"에서 "모델 하나 + 조건만 바꾸기"로.

---

## II. 데이터 — 무엇으로 학습하고 무엇으로 추론하는가

### 1. 훈련 데이터: **2D 이미지 풀 하나**
- 여러 소스에서 모은 2D 단면 이미지들.
- 같은 3D 샘플에서 나올 필요 없음. 서로 정렬될 필요 없음.
- 역할: "이 재료의 단면은 이렇게 생겨야 한다"는 **분포**를 모델이 배움.

### 2. 추론 조건: **(이미지, 위치) 쌍들**
- 사용자가 직접 얻은 단면(예: FIB).
- 각 단면 + 그 단면의 **위치(z 인덱스)**를 같이 입력.
- 장수·위치는 사용자 자유.

### 3. 3D 실측 데이터는 왜 필요 없는가
- 학습은 2D 풀에서 조건을 **실시간 합성**하는 방식으로 진행 (아래 III장 참조).
- 따라서 별도의 희소 3D 실측 데이터셋을 구축할 필요 없음.
- FIB 데이터는 오직 **추론 입력**으로만 쓰임.

> **한 줄**: "훈련은 흔한 2D 이미지 많이, 추론은 귀한 FIB 단면 몇 장."

---

## III. 학습 메커니즘 — 어떻게 배우는가

### 1. 조건 합성 방식
매 학습 step마다:
1. 풀에서 이미지 `X` 뽑음.
2. 랜덤 위치 `z*`에 `X`를 꽂아 **가짜 조건** 구성.
3. 조건 개수 `K`는 `0 ~ D-1` 중 랜덤 (0장/1장/여러 장 모드 동시 커버).
4. generator → 3D 출력.

### 2. 손실 함수의 역할 분담

| 손실 | 누가 | 무엇을 강제 |
|---|---|---|
| **recon loss** (L1) | generator | 조건 위치의 출력 슬라이스 = 조건 이미지 |
| **critic loss** (WGAN-GP) | 3개 critic (축별) | 축별 2D 슬라이스 분포가 실제 풀과 일치 |
| — (자연 귀결) | 3-축 critic | 인접 슬라이스 간 **3D 일관성** |

### 3. 조건 따르기가 보장되는 원리
- 학습 중 수많은 `(X, z*)` 페어에 대해 "mask=1인 곳은 입력값 그대로 출력"이라는 **identity mapping**을 학습.
- 추론 시 사용자가 `(X', z*')`를 넣으면 모델이 익힌 이 skill을 그대로 적용 → 출력의 `z*'` 슬라이스 ≈ `X'`.
- 별도의 inference-time 트릭이 아니라 **학습된 함수의 속성**.

> **한 줄**: "recon이 위치-이미지 매핑을, critic이 분포와 일관성을 책임."

---

## IV. 제약과 규칙

### 1. 조건축은 하나로 고정
- **물리적 이유**: FIB·마이크로톰 등 실제 슬라이싱은 **한 방향으로만** 자를 수 있음. 사용자 단면은 전부 같은 축에 수직.
- **설계 귀결**: `anchor.axis` 하나 고정. 학습·추론 모두 그 축에만 조건 놓음. 다른 두 축은 critic의 분포 제약만.

### 2. 조건 간 최소 간격 (`min_gap`)
- **왜**: 조건 2장이 바로 옆 위치에 있고 서로 무관한 이미지면 critic의 "부드러운 전이" 요구와 recon의 "이게 저기 저것" 요구가 **양립 불가** → 학습 품질 저하.
- **해결**: config에 `min_gap` 추가. 훈련 시 조건 위치를 `min_gap` 이상 떨어지게 rejection sampling. 추론 시에도 이 이상 권장.

> **한 줄**: "한 방향으로만 자르고, 조건끼리는 너무 붙이지 말 것."

---

## V. 추론 시 사용 방법

### 1. 세 가지 모드가 한 모델에 통합

| 조건 장수 | 동작 | 용도 |
|---|---|---|
| 0장 | vanilla SliceGAN처럼 (unconditional) | 랜덤 3D 생성 |
| 1장 | 그 이미지를 포함하는 3D | 최소 정보로 구조 유추 |
| 여러 장 | 모두 지키는 3D | FIB 복원 |

### 2. 공통 규칙
- 어느 경우든 **재학습 불필요**. 조건만 바꿔 바로 예측.
- 조건이 많을수록 원하는 구조에 가까운 결과.
- 조건 간 간격은 `min_gap` 이상으로 권장.
- 조건은 학습 시 정한 축에만 배치 가능.

> **한 줄**: "학습 한 번, 조건만 바꾸면 즉시 예측."

---

## VI. 미래 확장 — diffusion 백본 전환

이 설계의 핵심 장점은 **조건부 메커니즘이 생성기 종류와 독립적**이라는 점. 품질 개선이 필요할 때 백본만 교체할 수 있도록 구조를 유지.

### 1. 유지되는 것 (재사용)
- **데이터 파이프라인**: 2D 풀 + 조건 합성 로직 (`place_anchor_slices`, mask 채널).
- **조건화 API**: `(sparse, mask)` → 3D 출력 인터페이스.
- **config 체계**: `anchor.axis`, `min_gap`, 조건 개수 분포 등.

### 2. 교체되는 것
- `UNet3DGenerator` → `ConditionalDiffusionUNet3D`.
- critic 제거 가능 (diffusion이 분포 학습을 내재).
- recon loss는 **denoising loss로 흡수**되거나 classifier-free guidance로 대체.

### 3. 전환이 쉬운 이유
- 조건 입력 형식이 동일 → generator 내부만 바꾸면 됨.
- 평가 스크립트, 추론 CLI, 체크포인트 관리 등 주변 인프라는 그대로.

> **한 줄**: "지금은 GAN으로 빨리 검증, 나중에 diffusion으로 품질 업그레이드."

| 조건 개수 K | 0 ~ D-1 랜덤 | 0 ~ 자유 |
| 조건축 | `anchor.axis` 고정 | 같은 축 |
